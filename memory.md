# 项目记忆与开发日志 (Memory Log)

本文档记录了从项目初始化到当前状态的所有关键开发步骤、决策过程、遇到的问题及修复方案。

---

## 阶段一：项目启动与核心功能实现

**目标**：创建一个可以生成视频字幕的 Python 脚本。

1.  **用户需求**：生成视频字幕。
2.  **技术选型**：
    *   **识别引擎**：选择了 `faster-whisper`，因为它比 OpenAI 官方的 `whisper` 更快且内存占用更低。
    *   **脚本语言**：Python。
3.  **创建初始脚本 (`whisper_subtitle.py`)**：
    *   实现基本功能：接收视频文件路径，调用 `faster-whisper` 生成 SRT 字幕。
    *   添加了命令行参数 (`argparse`)，支持指定输入/输出文件、模型大小等。
4.  **遇到的问题**：
    *   **`float16` 兼容性**：在 Apple Silicon (Mac) 上执行时，因不支持 `float16` 计算类型而报错。
    *   **修复**：将脚本中的默认 `compute_type` 从 `float16` 修改为 `float32`，解决了在 Mac 上的运行问题。

---

## 阶段二：功能增强与体验优化

**目标**：提升脚本的实用性和用户体验。

1.  **新增功能**：
    *   **进度条与预估时间**：引入 `tqdm` 库，为识别、翻译、生成字幕等耗时操作添加了进度条。
    *   **多语言翻译**：集成了 `deep-translator` 库（使用免费的 Google Translate API），支持将识别出的字幕翻译成其他语言。
    *   **可配置输出路径**：添加了 `-d` / `--dir` 参数，允许用户指定字幕文件的输出目录。
    *   **高精度模式**：为了提升识别准确率，将默认识别模型从 `base` 升级到 `small`，并告知用户可以通过 `-m` 参数选用 `large` 等更高精度的模型。
    *   **指定开始时间**：增加了 `-s` / `--start` 参数，支持从视频的任意分钟开始生成字幕。

2.  **遇到的问题**：
    *   **`faster-whisper` 的语言参数**：`faster-whisper` 的自动语言检测需要传入 `None`，而不是字符串 `'auto'`。
    *   **修复**：在脚本中加入了逻辑判断，当用户输入 `'auto'` 时，自动转换成 `None` 再传给 `transcribe` 函数。

---

## 阶段三：性能极限优化

**目标**：解决生成速度慢的问题，提供多种加速方案。

1.  **速度优化 - Turbo 版**：
    *   创建 `whisper_subtitle_turbo.py`。
    *   **翻译加速**：将逐条翻译改为**批量翻译**，通过 `--batch-size` 参数控制批次大小，大幅减少网络请求次数，速度提升 5-10 倍。
    *   **生成加速**：将字幕生成逻辑从“逐条写入”改为“一次性写入”，避免了频繁的 I/O 操作，速度提升 10-20 倍。

2.  **准确率优化 - HD 高清版**：
    *   创建 `whisper_subtitle_hd.py`。
    *   **音频预处理**：在识别前，使用 FFmpeg 对音频进行**降噪 (afftdn)** 和 **音量响度标准化 (loudnorm)**，让模型“听”得更清。
    *   **高精度解码**：将 `beam_size` 提升到 5，并开启了**温度回退 (Temperature Fallback)**，以牺牲少量速度为代价，换取更高的识别准确率。

3.  **平台专属优化 - MLX 极速版 (Apple Silicon)**：
    *   创建 `whisper_subtitle_mlx.py`。
    *   **引入 MLX Whisper**：集成 Apple 官方的 MLX 框架，充分利用 Mac 的统一内存和 GPU，实现原生性能，速度比 `faster-whisper` 快 2-3 倍。
    *   **适配 API**：重写了模型加载和调用逻辑以适配 `mlx-whisper` 的 API。
    *   **遇到的问题**：`mlx-whisper` 同样不支持 `'auto'` 字符串作为语言参数。
    *   **修复**：在脚本中加入了相应的转换逻辑。

## 阶段四：视频合并与可视化界面

**目标**：实现完整的“视频进，带字幕的视频出”工作流，并提供图形化界面。

1.  **字幕合并脚本 (`merge_subtitle.py`)**：
    *   **技术选型**：使用强大的 `FFmpeg` 实现字幕硬编码。
    *   **功能**：支持自定义字幕字号、颜色、位置、描边等。
    *   **遇到的问题**：处理某些视频（特别是从 WMV 转码或时间戳损坏的视频）时，生成的视频**画面黑屏**。
    *   **分析**：原因是 FFmpeg 在处理时间戳 (PTS/DTS) 错误的视频流时，无法对齐画面和字幕，导致丢弃了几乎所有视频帧。
    *   **修复**：在 FFmpeg 命令中加入了 `-fflags +genpts` (强制生成时间戳) 和在滤镜中加入 `setpts=PTS-STARTPTS` (重置时间线)，彻底解决了黑屏问题。

2.  **Web UI 界面 (`webui.py`)**：
    *   **技术选型**：使用 `Gradio` 快速构建交互式 Web 界面。
    *   **功能**：
        *   提供“字幕生成”、“字幕合并”、“一键处理”三个功能选项卡。
        *   所有脚本的核心功能都集成为可交互的组件（如滑块、下拉菜单、复选框）。
        *   支持**实时流式输出**，用户可以在界面上看到识别和翻译的字幕逐条显示。
    *   **遇到的问题**：
        *   **依赖冲突**：新版 `gradio` 和 `huggingface_hub` 存在兼容性问题，导致 `ImportError`。
        *   **修复**：在 `requirements.txt` 中将 `gradio` 固定到 `4.44.0` 版本，`huggingface_hub` 固定到 `0.24.0` 版本。
        *   **代理/网络问题**：Gradio 在启动时因本地代理设置导致 `localhost` 无法访问而失败。
        *   **修复**：在 `webui.py` 脚本开头加入了自动清除代理环境变量的逻辑，并在 `launch()` 方法中强制使用 `127.0.0.1`，推荐用户使用 `--share` 参数。
